# Creditors Reconciliation

**Purpose**: To ensure the amount we think we owe (Creditors Ledger) matches what the supplier thinks we owe (Statement of Account).

## Common Differences

1. **Invoices not recorded**: Result -> We owe more. (Cr Creditors Ledger).
2. **Payments not received**: We paid, but they haven't received it yet. (Result -> Statement is too high).
3. **Returns not recorded**: We returned goods, but they haven't processed the credit note.
4. **Errors**: Calculation errors or wrong discounts.

## How to Answer

Usually a table format:

| No | Amount per Creditors Ledger | Amount per Statement |
| :--- | :--- | :--- |
| **Balance** | **10 000** | **12 000** |
| 1. Invoice omitted | + 500 | |
| 2. Payment not reflected | | - 2 000 |
| 3. Discount incorrect | - 100 | |
| **Final Balance** | **10 400** | **10 000** |

*Goal: The Final Balances must match.*

---

## Exam Tips

- **Reconcile to whom?**:
  - If **we** forgot an invoice, adjust the **Ledger**.
  - If **they** made a mistake (or haven't received money), adjust the **Statement**.
- **Credit Note**: Reduces the balance.
- **Invoice**: Increases the balance.
- **Interest**: Increases the balance.

---

## Quick Check

1. If a supplier charged us interest, do we Debut or Credit the Creditors Ledger? (Credit - Liability increases).
2. If we paid via EFT but it's not on the statement, where do we adjust? (Statement column).
3. What is the source document for a return? (Debit Note for us, Credit Note for them).
